disableBackwardSecondPass

Will not process the signal backwards as a second pass. Depending on the coefficients, the output may not be in phase with the input.

enableBackwardSecondPass

Will process the signal backwards as a second pass, using the same coeficients. This is to make sure the output remain in phase with the input

getOutput

Get the output signal

Returns

Float32Array :

the filtered signal

run

Launch the filtering. In the end, get the output using the method .getOutput()

setACoefficients(a)

Set the array of 'a' coefficients. Must be padded by an additional "1.0" because this set of coefficient will be addressed at it index "1" ( and not "0")

Parameters

  • Float32Array or Array a :

    the 'a' coeficients

setBCoefficients(b)

Set the array of 'b' coefficients

Parameters

  • Float32Array or Array b :

    the 'b' coeficients

setInput(signal)

Set the input signal. Will also reset the output to null.

Parameters

  • Float32Array signal :

    the signal
